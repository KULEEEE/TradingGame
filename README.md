# 📈 워크샵 모의 주식 트레이딩 게임

사내 워크샵용 실시간 모의 주식 게임. 빔프로젝터(메인 스크린) + 참가자 스마트폰으로 진행합니다.
단일 Node.js 서버에서 가격 시뮬레이션(GBM), 거래 체결, 리더보드가 모두 동작하며 외부 API/DB가 없습니다.

| 경로 | 화면 | 용도 |
|---|---|---|
| `/` | 참가자 (모바일) | QR 스캔 → 닉네임 입력 → 매수/매도 |
| `/screen` | 메인 스크린 | 프로젝터: 시세보드, 차트, 리더보드, QR, 타이머, 뉴스 티커 |
| `/admin` | 관리자 | 게임 제어, 종목/이벤트/뉴스, 참가자 관리 |

## 설치 & 로컬 실행

```bash
npm install
npm run dev        # = node server/index.js
```

서버가 뜨면 콘솔에 접속 URL이 출력됩니다 (LAN IP 자동 감지).
빌드 단계 없음 — 클라이언트는 정적 파일(vanilla JS + lightweight-charts)로 바로 서빙됩니다.

### 환경변수

| 변수 | 기본값 | 설명 |
|---|---|---|
| `PORT` | `3000` | 서버 포트 (`0.0.0.0` 바인딩) |
| `ADMIN_PASSWORD` | `admin1234` | 관리자 비밀번호 — **행사 전 반드시 변경** |
| `PUBLIC_URL` | (없음) | QR코드에 인코딩할 외부 URL. 없으면 LAN IP 자동 감지 |
| `TICK_SEC` | `10` | 가격 틱 간격(초). 캔들은 6틱(기본 1분봉)으로 집계 |

## 외부망 접속 (참가자가 모바일 데이터로 접속)

행사장 Wi-Fi를 신뢰할 수 없으므로 참가자는 각자 모바일 데이터로 접속하는 것이 기본 시나리오입니다. 두 가지 방법이 있습니다.

### 방법 A — Cloudflare 터널 (가장 빠름, 계정·배포 불필요)

진행자 노트북에서 서버를 돌리고, Cloudflare Quick Tunnel로 외부에 노출합니다.

```bash
# 최초 1회: cloudflared 설치
winget install Cloudflare.cloudflared    # macOS: brew install cloudflared

npm run dev:tunnel
```

몇 초 뒤 콘솔에 `https://랜덤이름.trycloudflare.com` 주소가 출력되고, **메인 스크린의 QR코드가 자동으로 이 주소로 갱신**됩니다. WebSocket도 그대로 통과합니다.

- 장점: 배포·계정 없이 즉시 사용. 노트북만 인터넷에 연결돼 있으면 됨
- 단점: URL이 실행할 때마다 바뀌고, 진행자 노트북이 꺼지면 게임도 꺼짐. 본 행사라면 방법 B 권장

### 방법 B — 클라우드 배포 (Railway, 안정적)

1. GitHub에 저장소 push 후 [railway.app](https://railway.app) → **New Project → Deploy from GitHub repo**
   (또는 CLI: `npm i -g @railway/cli && railway init && railway up`)
2. 자동으로 `npm install` + `npm start` 로 기동됩니다 (`railway.json` 포함됨).
3. **Settings → Networking → Generate Domain** 으로 공개 도메인 발급.
4. **Variables** 에 설정:
   - `PUBLIC_URL` = `https://발급받은도메인` ← QR코드가 이 주소를 인코딩 (**필수**)
   - `ADMIN_PASSWORD` = 행사용 비밀번호
5. Socket.IO는 WebSocket으로 동작하며 Railway에서 추가 설정 없이 지원됩니다.

> Render/Fly.io도 동일하게 `npm start` + 위 환경변수만 설정하면 됩니다.
> ⚠️ 클라우드의 디스크는 휘발성이라 `state.json` 스냅샷은 **프로세스 재시작**까지만 복구를 보장합니다 (재배포 시 초기화).

## 행사 당일 운영 절차

1. **사전 준비**
   - 클라우드 배포 확인: `https://도메인/healthz` → `{"ok":true}` (터널 운영 시 `npm run dev:tunnel` 후 출력된 주소로 확인)
   - (LAN 운영 시) 노트북·프로젝터·참가자가 **같은 Wi-Fi**에 있어야 하고, 공유기의 AP 격리(client isolation)가 꺼져 있어야 합니다.
2. 프로젝터 PC에서 `https://도메인/screen` 전체화면(F11). QR과 리더보드가 표시됩니다.
3. 진행자 폰/노트북에서 `/admin` 접속 → 비밀번호 로그인 → 초기자금/게임시간/수수료 설정.
4. 참가자들이 QR 스캔 → 닉네임 입력 (게임 중에도 늦은 참가 가능. 새로고침해도 토큰으로 자동 복구).
5. 관리자에서 **▶ 시작**. 진행 중 양념:
   - 종목별 **급등/급락**(강도·지속시간 선택), **즉시 점프 ±N%**
   - **시장 호황/폭락**, **뉴스 헤드라인 + 가격효과**
   - **거래정지/해제**, **상장폐지**(30초 카운트다운 — 보유자 탈출 찬스) / 즉시 상폐
   - **⏸ 서킷브레이커**: 가격·거래·타이머 전부 동결 (진행자 설명 타임)
   - **🧹 뉴스 전체 삭제**: 모든 화면의 뉴스 티커를 즉시 비움
6. 시간이 다 되면 자동 종료(또는 **🏁 종료**) → 메인 스크린에 시상식 표시 (1~3등 강조).
7. 한 판 더 하려면 **🔄 리셋** (참가자 계정은 유지, 잔고/가격 초기화).

### 서버가 죽었을 때

5초마다 전체 상태가 `state.json` 에 저장됩니다. 같은 머신에서 서버를 다시 켜면 참가자/잔고/가격이 복구되고,
진행 중이던 게임은 **일시정지 상태**로 복구되므로 관리자에서 **▶ 재개**만 누르면 됩니다.

## 게임 규칙

- 가격: 종목별 GBM(기하 브라운 운동), **10초 1틱**(기본, `TICK_SEC`로 변경 가능). 캔들은 6틱 = 1분봉 OHLC (종목당 최근 360개)
- σ/μ 설정값은 **초당 기준**이며 서버가 틱 간격에 맞게 μ·dt, σ·√dt 로 스케일 — 틱 간격을 바꿔도 분당 변동성은 동일
- 모든 가격 개입은 **modifier 스택** `{driftDelta, volMul, ticksLeft}` 으로 통일 — 매 틱 만료분 제거 후 기본 σ/μ에 합산
- 뉴스 티커는 최신 5건, 2분 지난 뉴스는 자동으로 사라짐
- 시장가 즉시 체결, 공매도 없음, 잔고/보유수량 서버 검증, 잔고 갱신은 단일 스레드에서 원자적 처리
- 게임 시작 전·서킷브레이커 중·거래정지/상폐 종목 거래 불가 (상폐 **카운트다운 중에는** 탈출 매도 가능)
- 우승: 종료 시점 **총자산(현금 + 보유주식 평가액)** 1위. 상폐 종목 평가액은 0원
- 등락률 색상은 한국식 (상승 빨강 / 하락 파랑), 금액은 원화 콤마 표기

## 프로젝트 구조

```
├── config/stocks.json     # 기본 종목 8개 (admin에서 추가/수정 가능)
├── shared/events.js       # ★ Socket.IO 이벤트 이름 + 페이로드 명세 (서버/클라 공유)
├── server/
│   ├── index.js           # Express + Socket.IO 부트스트랩, 라우팅, QR, LAN IP
│   ├── game.js            # 게임 엔진: GBM 틱, modifier, 캔들, 거래, 상폐, 리더보드
│   └── persist.js         # state.json 5초 스냅샷 (비동기 tmp+rename)
├── public/                # 빌드 없는 정적 클라이언트
│   ├── index.html  + js/app.js     # 참가자
│   ├── screen.html + js/screen.js  # 메인 스크린
│   ├── admin.html  + js/admin.js   # 관리자
│   └── js/common.js, css/          # 공통 유틸/테마
└── scripts/verify.js      # E2E 검증 (npm run verify)
```

## 검증

서버를 켜둔 상태에서:

```bash
npm run verify   # 참가→매매→틱→리더보드→이벤트→서킷브레이커→복구→종료→리셋 47개 체크
```

## 성능 메모

- 틱 브로드캐스트는 `{ts, prices:{SYM:가격}}` 변경분만 전송 (종목 8개 기준 수백 바이트/틱)
- 차트 히스토리는 접속/종목 선택 시 1회만 전송, 이후 현재 봉은 클라이언트에서 틱으로 집계
- 동시 50명 + 10초 틱 기준 매우 여유 (리더보드는 틱마다 + 거래 직후 500ms 디바운스로 집계)
- 스냅샷 저장은 비동기 쓰기 → 틱 루프 비블로킹
