# 📈 실시간 모의 주식 트레이딩 게임

가상 자금으로 실시간 시세에 따라 주식을 사고팔며 수익률을 겨루는 게임입니다.
Node.js 서버 하나로 가격 시뮬레이션, 거래 체결, 리더보드가 모두 동작하며 외부 API나 DB가 필요 없습니다.

화면은 세 가지이고, 접속 주소 뒤에 무엇을 붙이느냐로 구분됩니다.

| 접속 방법 | 화면 |
|---|---|
| 주소 그대로 접속 | **사용자 화면** (모바일) — 매수/매도 |
| 주소 뒤에 `/admin` 붙이기 | **관리자 화면** — 게임 제어 |
| 주소 뒤에 `/screen` 붙이기 | **메인 스크린** (선택) — 시세보드, 리더보드, QR. 프로젝터·TV용 |

예: 서버 주소가 `http://192.168.0.10:3000`이면 관리자 화면은 `http://192.168.0.10:3000/admin` 입니다.

## 필수 설치

| 프로그램 | 용도 | 설치 |
|---|---|---|
| [Node.js](https://nodejs.org/ko/download) (18+) | 게임 서버 실행 | 공식 사이트에서 LTS 버전 설치 |
| [cloudflared](https://developers.cloudflare.com/cloudflare-one/connections/connect-networks/downloads/) | 외부 접속용 Cloudflare 터널 | `winget install Cloudflare.cloudflared` (macOS: `brew install cloudflared`) |

cloudflared는 계정 가입 없이 설치만 하면 됩니다.

## 실행 방법

```bash
npm install        # 최초 1회
npm run dev:tunnel
```

몇 초 뒤 콘솔에 `https://랜덤이름.trycloudflare.com` 주소가 출력됩니다. 이 주소로 어디서든(모바일 데이터 포함) 접속할 수 있고, 메인 스크린의 QR코드도 자동으로 이 주소를 가리킵니다. 빌드 단계는 없습니다.

참고:
- 이 주소는 실행할 때마다 바뀌고, 서버를 돌리는 컴퓨터가 꺼지면 게임도 함께 꺼집니다.
- 같은 Wi-Fi(LAN)에서만 쓸 거라면 터널 없이 `npm run dev`로 실행해도 됩니다. 콘솔에 출력되는 LAN IP 주소로 접속하세요.

### 환경변수

| 변수 | 기본값 | 설명 |
|---|---|---|
| `PORT` | `3000` | 서버 포트 |
| `ADMIN_PASSWORD` | `admin1234` | 관리자 비밀번호 — **반드시 변경 권장** |
| `PUBLIC_URL` | (없음) | QR코드에 인코딩할 외부 URL. 없으면 자동 감지 |
| `TICK_SEC` | `10` | 가격 틱 간격(초) |

## 사용법 — 사용자 화면

1. 접속 후 닉네임을 입력하고 **참가하기** (게임 도중에도 참가 가능, 새로고침해도 자동 복구).
2. 상단에 내 **총자산 / 현금 / 수익률 / 순위**가 실시간으로 표시됩니다.
   <img width="320" alt="image" src="https://github.com/user-attachments/assets/11223506-6e21-4db7-a317-0f1eb0359032" />
3. 하단 탭 3개로 이동합니다.
   - **📊 시세** — 전체 종목의 현재가와 등락률. 종목을 누르면 상세 화면(차트 + 주문)이 열립니다.
   - **💼 포트폴리오** — 보유 종목, 평가액, 손익.
   - **📋 내역** — 내 거래 기록.
4. 종목 상세에서 **매수/매도** 선택 → 수량 입력(**최대** 버튼으로 풀매수/풀매도) → 주문. 시장가로 즉시 체결됩니다.
   <img width="320" alt="image" src="https://github.com/user-attachments/assets/f721d81a-5270-4f8c-86c1-35b77e1fc714" />
5. 게임이 끝나면 최종 순위와 내 결과가 표시됩니다. 우승 기준은 종료 시점 **총자산(현금 + 보유주식 평가액)**.

거래가 안 될 때: 게임 시작 전, 서킷브레이커 중, 거래정지/상장폐지 종목은 거래할 수 없습니다 (상폐 카운트다운 중에는 매도만 가능).

## 사용법 — 관리자 화면

주소 뒤에 `/admin`을 붙여 접속한 뒤 비밀번호(`ADMIN_PASSWORD`)로 로그인하면 카드별로 기능이 나뉩니다.

### 게임 제어
- **▶ 시작 / 🏁 종료 / 🔄 리셋** — 리셋은 참가자 계정은 유지하고 잔고·가격만 초기화 (한 판 더).
- **⏸ 서킷브레이커 / ▶ 재개** — 가격·거래·타이머를 전부 동결했다가 다시 진행.
- **초기자금 / 게임시간(분) / 수수료율(%)** 설정 후 **설정 적용**.

### 시장 이벤트 · 뉴스
- **🚀 시장 호황 / 📉 시장 폭락** — 전 종목에 상승/하락 압력. 강도(약/중/강)와 지속시간(15/30/60초) 선택.
- **📰 뉴스 발송** — 헤드라인을 입력해 모든 화면의 뉴스 티커에 표시. 종목과 급등/급락을 함께 선택하면 해당 종목 가격에도 효과가 적용됩니다.
- **🧹 뉴스 전체 삭제** — 모든 화면의 뉴스 티커를 즉시 비움.

### 종목 관리
종목별로 한 줄씩 제어합니다.
- **급등/급락** — 선택한 강도·지속시간으로 해당 종목에 드리프트 적용.
- **즉시 점프** — 가격을 즉시 ±N% 이동.
- **정지/해제** — 해당 종목 거래 중지.
- **상폐** — 30초 카운트다운 후 상장폐지 (보유자는 그동안 매도로 탈출 가능). 즉시 상폐도 가능.
- **σ(변동성) / μ(드리프트)** 수정, 하단에서 **➕ 종목 추가** (심볼·종목명·시작가·σ·μ).

### 참가자
순위·총자산·수익률·접속 상태를 확인하고, 필요하면 **강퇴**할 수 있습니다.

## 기타

- 서버가 죽어도 5초마다 `state.json`에 상태가 저장되므로, 같은 머신에서 다시 켜면 참가자/잔고/가격이 복구됩니다. 진행 중이던 게임은 일시정지 상태로 복구되니 관리자에서 **▶ 재개**만 누르면 됩니다.
- 검증: 서버를 켜둔 상태에서 `npm run verify` (참가→매매→틱→이벤트→복구→종료→리셋 E2E 체크).
